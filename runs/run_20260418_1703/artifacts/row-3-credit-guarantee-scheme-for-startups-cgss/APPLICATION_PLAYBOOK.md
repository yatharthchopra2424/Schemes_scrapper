# Application Playbook
## How to Use This File
For each new client case, assign it a Case ID, pick the target scheme, and follow the steps below.

## Stage 1: Eligibility Verification (Day 1–3)
### Checklist
The following checklist is based on the actual eligibility criteria for the Credit Guarantee Scheme for Startups (CGSS):
✅ The startup should be recognized by DPIIT as per Gazette Notifications issued from time to time.
✅ The startup should not be in default to any lending/investing institution and not classified as Non-Performing Asset as per RBI guidelines.
✅ The startup should have a valid business plan and financial statements.
✅ The startup should meet the definition of a startup as per the DPIIT notification.

## Stage 2: Document Collection (Day 3–10)
### Universal Documents
The following documents are required for all schemes:
* PAN
* ITR
* GST
* Bank statements
* Udyam registration

### Scheme-Specific Documents
The following documents are required for the CGSS:
* DPIIT recognition certificate
* Incorporation certificate
* KYC documents
* Business plan
* Financial statements

### Document Quality Check
The following checklist is used to verify the quality of the documents:
* File size and format
* Name matching with the startup's name
* Certification requirements
* Language (English)

### Client Communication Template
The following message can be used to communicate with the client:
"Dear [Client], we are applying for the Credit Guarantee Scheme for Startups (CGSS) on your behalf. Please find attached the required documents and instructions. The application portal is https://jansamarth.in/. The deadline for submission is [Deadline]. If you have any questions or concerns, please do not hesitate to reach out to us."

## Stage 3: Application Drafting (Day 10–18)
### Project Report / DPR
A Project Report / DPR is not required for the CGSS.

### Application Form Filling Protocol
The following protocol is used to fill the application form:
1. Ensure all required documents are attached.
2. Fill in all fields accurately and completely.
3. Verify the startup's eligibility criteria.
4. Ensure the business plan and financial statements are up-to-date.
5. Review the application form for any errors or omissions.

### Internal Review Gate
The following checklist is used to review the application form:
* All fields are complete and accurate.
* All required documents are attached.
* The startup meets the eligibility criteria.
* The business plan and financial statements are up-to-date.

## Stage 4: Submission & Follow-Up (Day 18–30)
### Submission Protocol
The following protocol is used to submit the application form:
1. Submit the application form through the Jan Samarth Portal.
2. Take a screenshot of the submission confirmation.
3. Store the reference number for future reference.
4. Set follow-up reminders for the next 30 days.

### Follow-Up Script
The following script can be used to follow up with the nodal officer or portal helpdesk:
"Dear [Nodal Officer/Helpdesk], we submitted an application for the Credit Guarantee Scheme for Startups (CGSS) on [Date] with reference number [Reference Number]. We would like to inquire about the status of our application. Please let us know if any additional documents or information are required. Thank you for your assistance."

### Escalation Path
The following escalation path can be used:
- Level 1: Portal helpdesk (https://jansamarth.in/)
- Level 2: Nodal officer (dipiit.startupindia@gov.in, 1800-11-5565)
- Level 3: RTI filing if no response within statutory SLA

## Stage 5: Post-Approval (Day 30–90+)
### Approval Actions
The following actions are taken after approval:
* Verify the sanctioned amount.
* Review the terms and conditions of the guarantee.
* Ensure the startup understands its obligations and responsibilities.
* Follow up with the client to ensure successful utilization of the guarantee.
