# Application Playbook

## How to Use This File
For each new client case, assign it a Case ID, pick the target scheme, and follow the steps below.

---

## Stage 1: Eligibility Verification (Day 1–3)

### Checklist
- [ ] Startup or entrepreneur working in emerging technologies (AI, IoT, Blockchain, FinTech, MedTech, AgriTech, Cybersecurity, AR/VR, or other domain-specific areas)  
- [ ] Engaged in technology-driven product development  
- [ ] Demonstrates potential for scalability and market impact  
- [ ] Open to applicants from Tier-2 and Tier-3 cities (promoting inclusive growth)  
- [ ] No specific registration type, turnover threshold, employee count, or age of business mandated (flexible for early-stage startups)  
- [ ] Must identify relevant Centre of Entrepreneurship (CoE) based on technology domain and location  

---

## Stage 2: Document Collection (Day 3–10)

### Universal Documents (always required)
- Certificate of Incorporation or registration (for registered entities)  
- PAN card of the entity or promoters  
- Bank account details  
- Udyam Registration (if applicable)  
- GST registration (if applicable)  
- ITR of last 2–3 years (if available)  
- Audited financial statements (if available)  

### Scheme-Specific Documents
- Business plan or project proposal  
- Team details and resumes  
- Proof of concept or prototype (if available)  
- Any existing funding or investment details  
- Intellectual property details (if applicable)  

### Document Quality Check
- All documents must be legible, scanned in PDF format  
- File size should not exceed 5 MB per document  
- Names on documents must match the applicant entity or promoters  
- Self-attested copies acceptable; notarization not required unless specified by CoE  
- Documents must be in English or Hindi; translations required for other languages

## Stage 3: Application Drafting (Day 10–18)

### Project Report / DPR (if required)
Not explicitly required; however, a detailed **business plan or project proposal** is mandatory. If the CoE or selection committee requests a DPR, verify with the nodal officer. Standard sections to include: executive summary, problem statement, solution, technology, market analysis, business model, team, financial projections, and implementation roadmap.

### Application Form Filling Protocol
1. Access the application via the STPI portal (https://stpi.in/) or the designated CoE channel.  
2. Fill all fields accurately; do not leave mandatory sections blank.  
3. Upload documents in PDF format, clearly labeled (e.g., “Business_Plan.pdf”, “Team_Resumes.pdf”).  
4. Ensure the business plan highlights innovation, scalability, and market impact in emerging technologies.  
5. Double-check that PAN, bank details, and incorporation certificate match the applicant entity.  
6. Select the correct technology domain and preferred CoE location during form submission.  
7. Declarations and consent boxes must be checked before final submission.  
8. Save a copy of the submitted application and note the application reference number.  

### Internal Review Gate
- [ ] All application form fields completed  
- [ ] Business plan/project proposal uploaded and reviewed  
- [ ] Team details and resumes attached  
- [ ] Proof of concept/prototype (if available) uploaded  
- [ ] Certificate of Incorporation/registration submitted  
- [ ] PAN card and bank details provided  
- [ ] Existing funding/investment details disclosed (if any)  
- [ ] Intellectual property details included (if applicable)  
- [ ] All documents legible, correctly named, and within size limits  
- [ ] Declarations and consent checkboxes selected  
- [ ] Application reference number saved and shared with client

## Stage 4: Submission & Follow-Up (Day 18–30)

### Submission Protocol
- Submit the application via the STPI portal (https://stpi.in/) or designated CoE channel  
- Take a screenshot of the confirmation page showing application reference number and timestamp  
- Store the reference number securely in the case file  
- Set calendar reminders for follow-up at 15 days and 30 days post-submission
