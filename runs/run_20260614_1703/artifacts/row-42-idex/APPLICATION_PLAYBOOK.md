# Application Playbook

## How to Use This File
For each new client case, assign it a Case ID, pick the target scheme, and follow the steps below.

---

## Stage 1: Eligibility Verification (Day 1–3)

### Checklist
- [ ] Entity is a startup, MSME, innovator, or R&D institute registered in India  
- [ ] Entity is working on innovative defence and aerospace technologies  
- [ ] Proposal addresses a specific problem statement released by armed forces, DPSUs, or OFB  
- [ ] Entity has valid Certificate of Incorporation/Registration  
- [ ] Entity has a valid PAN  
- [ ] Entity has GST registration (if applicable)  
- [ ] Entity can provide detailed project proposal, technical specifications, budget, team credentials, and prior work/prototypes  

---

## Stage 2: Document Collection (Day 3–10)

### Universal Documents (always required)
- Certificate of Incorporation/Registration  
- PAN of the entity  
- GST registration certificate (if applicable)  
- Bank statements (last 6 months)  
- Udyam Registration Certificate (for MSMEs)  
- ITR of the entity for last 2 financial years  
- Audited financial statements (if applicable)  

### Scheme-Specific Documents
- Detailed project proposal addressing a specific iDEX problem statement  
- Technical specifications and innovation details  
- Budget and financial plan (milestone-based)  
- Team credentials and expertise (CVs of key personnel)  
- Any prior work or prototypes related to the proposal  

### Document Quality Check
- All documents must be legible, scanned in PDF format  
- File size for each document must not exceed 5 MB  
- File names must follow format: `SchemeName_DocumentType_EntityName_Date.pdf` (e.g., `iDEX_Proposal_Innovatech_20240515.pdf`)  
- All certificates must be valid and not expired  
- Documents in regional languages must be accompanied by certified English/Hindi translation  
- Project proposal must clearly map to a currently active problem statement on the iDEX portal

## Stage 3: Application Drafting (Day 10–18)

### Project Report / DPR (if required)
A **Detailed Project Proposal** is required (not a traditional DPR). It must include:  
- Problem statement being addressed (copied from iDEX portal)  
- Innovation objective and technical approach  
- Methodology and work plan  
- Milestone-wise timeline and deliverables  
- Budget breakdown (capital, operational, manpower, contingencies)  
- Expected outcomes and alignment with defence priorities  
- Risk assessment and mitigation plan  
- Team qualifications and roles  
- Any prior work, prototypes, or IPR status  

### Application Form Filling Protocol
1. Register as an innovator on the iDEX portal (https://www.idex.gov.in/) before initiating application  
2. Select a currently active problem statement from the “Problem Statements” section  
3. Ensure the proposal title and objective clearly reference the selected problem statement  
4. Fill all fields in the online proposal form — do not leave any mandatory section blank  
5. Upload all scheme-specific documents as per the prescribed format and size limits  
6. Double-check that the budget summary matches the detailed financial plan uploaded  
7. Confirm that team credentials highlight relevant defence/aerospace experience  
8. Review the declaration section carefully — ensure authorized signatory has access to digital signature  

### Internal Review Gate
- [ ] All universal and scheme-specific documents are attached and clearly labeled  
- [ ] Project proposal addresses an active problem statement from the iDEX portal  
- [ ] Budget is realistic, milestone-linked, and does not exceed INR 1.5 Crore  
- [ ] Technical specifications demonstrate novelty and defence relevance  
- [ ] Team CVs show expertise in relevant technology domains  
- [ ] Declarations are signed and dated by authorized signatory  
- [ ] No plagiarism or generic content — proposal is tailored to the specific problem statement  
- [ ] Application is submitted well before any internal deadline to allow for revisions

## Stage 4: Submission & Follow-Up (Day 18–30)

### Submission Protocol
- Take a screenshot of the final submission confirmation page showing Application ID and timestamp  
- Save the Application ID in the case file and set up a tracking spreadsheet  
- Store all submitted documents in a dedicated folder named: `iDEX_[EntityName]_ApplicationID_[Date]`  
- Set follow-up reminders for:  
  - 7 days post-submission (to confirm receipt)  
  - 15 days post-submission (to check evaluation status)  
  - Every 10 days thereafter until outcome is communicated

## Stage 5: Post-Approval (Day 30–90+)

### Approval Actions
- [ ] Obtain and verify the official approval letter from iDEX/DIO specifying sanctioned amount and project duration  
- [ ] Confirm the milestone-based disbursement schedule and conditions for each tranche  
- [ ] Set up a dedicated project bank account (if required) for tracking grant utilization  
- [ ] Develop a utilization plan aligned with approved budget and milestones  
- [ ] Prepare for submission of periodic progress reports and financial statements as per iDEX guidelines  
- [ ] Coordinate with defence testing facilities for trials, if applicable to the project  
- [ ] Ensure IPR documentation is in place, noting government’s rights for defence use  
- [ ] Track milestone achievement and trigger invoicing for success fee as per consultancy agreement  
- [ ] Maintain audit-ready records of all expenditures, communications, and deliverables  
- [ ] Prepare utilization certificate and final report for submission upon project completion  
- [ ] Explore opportunities for induction into service or partnership with defence PSUs/OFBs post-completion
