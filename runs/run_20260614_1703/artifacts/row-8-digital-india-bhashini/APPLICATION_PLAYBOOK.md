# Application Playbook

## How to Use This File
For each new client case, assign it a Case ID, pick the target scheme, and follow the steps below.

---

## Stage 1: Eligibility Verification (Day 1–3)

### Checklist
- [ ] Entity type: Startup (DPIIT recognized or self-declared)  
- [ ] Entity type: Government agency (Central/State/UT department or undertaking)  
- [ ] Entity type: Citizen (individual seeking linguistic access/services)  
- [ ] Willingness to collaborate on linguistic technology innovation (for startups/agencies)  
- [ ] Alignment with objectives: Facilitating seamless communication between Indian languages and English, celebrating linguistic diversity, fostering startup-government collaboration, preserving linguistic diversity through technology  
- [ ] No financial eligibility thresholds (turnover, investment, etc.) specified — participation is collaboration-based  
- [ ] Pan-India applicability — no geographic restrictions  
- [ ] No minimum age of business or employee count requirements specified  
- [ ] Sector: Open to all sectors, with focus on language technology, AI, NLP, digital inclusion  

> **Note**: Eligibility is based on collaboration intent rather than formal criteria. Verify with nodal officer for specific partnership terms.  
> **Sources**:  
> - Scheme Key Facts: Eligibility, Target Beneficiaries, Objectives  
> - Supporting Evidence: https://digitalindia.gov.in/initiative/digital-india-bhashini-2

## Stage 2: Document Collection (Day 3–10)

### Universal Documents (always required)
- [ ] Incorporation/Registration Certificate (for startups/agencies) or ID proof (for citizens)  
- [ ] PAN Card  
- [ ] GST Registration Certificate (if applicable)  
- [ ] Udyam Registration (for MSMEs/startups)  
- [ ] Bank Statement (last 3 months)  
- [ ] ITR (last 2 years, if applicable)  
- [ ] Authorization Letter (if applying via representative)  

### Scheme-Specific Documents
- [ ] Project Proposal outlining linguistic technology solution or collaboration idea (for startups/agencies)  
- [ ] Proof of DPIIT recognition (for startups claiming startup status)  
- [ ] Language resource details (datasets, models, tools) if proposing technological contribution  
- [ ] Letters of intent or collaboration agreements with government agencies (if applicable)  
- [ ] Use case description for citizens seeking linguistic access (e.g., need for translation in specific service)  

### Document Quality Check
- [ ] All documents are legible scans or certified PDFs  
- [ ] File names match the document type (e.g., PAN_Card.pdf, Udyam_Registration.pdf)  
- [ ] No password protection or encryption on files  
- [ ] Documents are in English or Hindi; if in regional language, include certified translation  
- [ ] File size < 10 MB per document; compress if necessary  
- [ ] Signatures and seals are visible and valid (where applicable)  
- [ ] Dates on documents are current (e.g., bank statements < 3 months old, ITR < 2 years old)

## Stage 3: Application Drafting (Day 10–18)

### Project Report / DPR (if required)
- **DPR Required?** Yes, in the form of a collaboration proposal or project report.  
- **Standard Sections**:  
  1. Executive Summary  
  2. Objectives and Alignment with Bhashini Goals  
  3. Linguistic Technology Solution (e.g., ASR, MT, TTS, OCR in Indian languages)  
  4. Innovation and Novelty Aspect  
  5. Implementation Plan and Timeline  
  6. Resource Requirements (datasets, manpower, infrastructure)  
  7. Expected Outcomes and Impact (linguistic coverage, user base, scalability)  
  8. Collaboration Model (with startups, agencies, or citizens)  
  9. Sustainability and Scalability Plan  
  10. Budget (if seeking in-kind support or detailing self-funded effort)  
- **Note**: Since no direct funding is mentioned, the DPR focuses on collaboration intent and technical merit. Verify format with nodal officer.  
> **Sources**: Scheme Key Facts – Objectives, Benefits; Supporting Evidence: https://digitalindia.gov.in/initiative/digital-india-bhashini-2  

### Application Form Filling Protocol
1. Access the collaboration portal via Digital India Corporation (DIC) or MeitY coordination channel — no standalone public form is currently published.  
2. Clearly state the entity type (startup/government agency/citizen) and proposed role (technology provider, collaborator, end-user).  
3. Describe the linguistic technology contribution (e.g., dataset, model, tool) or service need in precise terms.  
4. Highlight innovation, scalability, and alignment with Bhashini’s vision of linguistic inclusivity.  
5. Attach all supporting documents as per the checklist.  
6. Ensure the proposal emphasizes collaboration, not financial assistance.  
7. Double-check that all claims about technology capability are verifiable (avoid overpromising).  
8. Submit via email or designated portal as advised by the nodal officer; retain acknowledgment.  
> **Note**: No public application form is evidenced. Protocol based on standard innovation collaboration practice.  
> **Sources**: Scheme Key Facts – Application Process, Key Caveats; Supporting Evidence: https://digitalindia.gov.in/initiative/digital-india-bhashini-2  

### Internal Review Gate
- [ ] All client details and entity information are correctly entered  
- [ ] Collaboration proposal is complete, technically sound, and aligned with Bhashini objectives  
- [ ] All universal and scheme-specific documents are attached and validated  
- [ ] Declarations regarding originality of IP, data privacy, and collaboration intent are included  
- [ ] No false claims about funding or guaranteed benefits are made  
- [ ] File names and formats meet quality standards  
- [ ] Proposal is reviewed for clarity, feasibility, and impact  
- [ ] Final sign-off by senior consultant before submission

## Stage 4: Submission & Follow-Up (Day 18–30)

### Submission Protocol
- [ ] Submit the collaboration proposal via the designated channel (email to DIC/MeitY or portal as advised)  
- [ ] Obtain and save acknowledgment receipt (email confirmation or portal reference number)  
- [ ] Store all submitted documents in a secure, labeled folder (Case ID_Digital India Bhashini_Submission)  
- [ ] Set follow-up reminders:  
  - Initial follow-up: 7 days after submission  
  - Second follow-up: 14 days after submission  
  - Escalation trigger: 30 days with no response

## Stage 5: Post-Approval (Day 30–90+)

### Approval Actions
- [ ] Obtain formal collaboration acknowledgment or MoU from Digital India Corporation (DIC) / MeitY  
- [ ] Verify sanctioned support (if any): access to datasets, cloud credits, mentorship, or pilot opportunity (note: no direct financial grant is evidenced)  
- [ ] If applicable, sign Non-Disclosure Agreement (NDA) or IP sharing agreement  
- [ ] Initiate project execution as per approved collaboration plan  
- [ ] Maintain records of milestones, deliverables, and utilization of resources  
- [ ] Prepare utilization certificate or progress report as required by DIC/MeitY  
- [ ] Coordinate with consultant for success fee invoicing upon milestone achievement (if applicable)  
- [ ] Explore scaling opportunities or follow-on collaborations post-pilot  

> **Note**: Since no financial disbursement is mentioned, “approval” refers to collaboration acceptance. Actions focus on project execution and reporting.  
> **Sources**: Scheme Key Facts – Financial Support, Key Caveats; Supporting Evidence: https://digitalindia.gov.in/initiative/digital-india-bhashini-2
